# 前两天重点复习

#### constructor

> 原生的原型对象中都函数这个属性，这个属性指向对应的构造函数

#### prototoype

> 所有的函数都有这个属性，这个属性指向对应的原型对象（显式原型）

#### __proto__

> 所有的对象都有这个属性，这个属性指向该对象直接继承的对象（隐式原型）

#### 面向对象的概念

> 凡是不用亲力亲为，支配其他对象解决问题即可。
面向对象仍然离不开面向过程。
通常面向对象应用在大型应用软件开发，大型项目使用面向对象的思想会比较爽。

#### 构造函数的概念

> 就是用来构造（创建）对象的函数。
在js中，构造函数和普通函数是一个东西，只不过使用方式不同而已。
通常构造函数我们让它的名字首字母大写。
同时，在js中，构造函数也可以说是类。

#### 类的概念

> 就是对某一些具有相同特征和特性的对象的抽象描述。
类可以看作是一个模版，专门用来生产对象的，
这和构造函数的功能一样，所以类就是构造函数。
注意：类、构造函数不是工厂函数。

#### 实例的概念

> 通过构造函数(类)创建的对象就叫这个构造函数(类)的实例。
实例就是实实在在的对象，不那么抽象而已。

#### 实例的类型

> 实例的类型就是对应的构造函数(类)的名字。

#### 原型的概念

> 任何函数在诞生的时候，都会跟着诞生一个原型对象，简称原型;
函数会有一个属性prototype指向这个原型对象;
原型对象会有一个constructor属性指向函数。

#### 原型的作用

> 如果把函数当做构造函数来使用，那么创建的实例就会继承这个原型对象。

#### 原型对象和原型属性（显式原型和隐式原型）

> 显式原型和隐式原型说的都是原型对象。
显式原型说的是函数的prototype属性，
隐式原型说的是实例的__proto__属性。
函数既有显式原型又有隐式原型。

#### 原型链的概念  ==> 原型链长度问题 

1. 所有的对象有都有原型对象，原型对象还有原型对象，
一直到Object.prototype为止，整个原型结构被形象的称之为原型链。
2. 从对象开始，由__proto__连接起来的整个原型(继承)结构，称之为原型链。
3. 原型链的长度可长可短。

#### 内置对象原型链结构

1. {} ==> Object.prototype ==> null
2. [] ==> Array.prototype ==> Object.prototype ==> null
3. new Person ==> Person.prototype ==> Object.prototype ==> null
4. Function ==> Function.prototype ==> Object.prototype ==> null
5. Object ==> Function.prototype ==> Object.prototype ==> null

#### 继承的概念

> 对象可以使用本不属于的自己属性与方法，这就是继承。
